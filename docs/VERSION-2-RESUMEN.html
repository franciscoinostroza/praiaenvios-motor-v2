Formatos Combinados Praia + UPS

Documentación técnica de la Versión 2 — Integración de UPS Internacional al motor de cotizaciones de Praia Envíos.

Opción 3 · Feature Flag · Formato Completo

📋Resumen

La Versión 2 incorpora UPS Internacional al motor de cotizaciones de Praia Envíos, permitiendo mostrar ambas opciones (Praia y UPS) en un mismo mensaje para Venezuela, o solo UPS para otros países.

Un feature flag FORMATO_COMPLETO permite activar/desactivar el nuevo flujo sin afectar a usuarios existentes. Con false (default), el sistema se comporta exactamente como la versión legacy.

🔀Flujo General

POST /cotizar → FORMATO_COMPLETO=true? → NO → Legacy: solo Praia (igual que antes) → SÍ → ¿País = Venezuela? → SÍ → Praia + UPS en PARALELO (Promise.allSettled, timeout 8s) → UPS ok → mensaje único con Praia + UPS → UPS falla → solo Praia con aviso → NO → ¿País internacional? → SÍ → solo UPS con 1 reintento → NO → Brasil → legacy (solo Praia)

🗄️Nuevas Tablas en DB

Almacena 37 ciudades LATAM con información para el trecho nacional hasta Curitiba.

Columna Descripción
ciudadNombre de la ciudad
codigo_iataCódigo IATA del aeropuerto

EDITABLE DESDE ADMIN Permite agregar/quitar ciudades sin deploy.

Almacena configuraciones de texto clave-valor.

Columna Descripción

✅Validaciones Nuevas

codigo_postal_destino es obligatorio para cotizaciones UPS. Si no se envía, la cotización UPS devuelve error de validación.

💬Whapify — Sin Cambios

Usuario WhatsApp → Whapify (conversación) → Whapify pide datos paso a paso → Cuando tiene todo → Whapify llama POST /whapify/cotizar → Nosotros: mismo handler que /cotizar → Devolvemos resultado_final + mensaje_formateado → Whapify envía mensaje al usuario

La integración con Whapify no cambió. Solo cambia el contenido del mensaje devuelto (ahora puede incluir UPS).
